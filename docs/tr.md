# Техническое решение проекта «Название проекта»

## Введение
- **Цель проекта:**  
  Проект направлен на создание многофункционального социально-гибридного приложения. Его ядром является живая карта с реальным временем, вокруг которой строится вся социальная экосистема для друзей, единомышленников и сообществ. Это не просто карта с друзьями, а полноценная социальная сеть.

- **Основания для разработки:**  
  Учебный проект для личного опыта и развития.

## Глоссарий
| Термин | Определение |
|---|---|
| Пользователь | Человек, зарегистрированный в приложении, который может общаться, добавлять друзей и создавать контенТ. |
| Профиль | Личная страница пользователя с аватаром, именем, статусом и основной информацией. |
| Друг | Пользователь, с которым установлена двусторонняя связь через систему запросов, дающая доступ к дополнительным функциям общения. |
| Сообщение | Текстовое, голосовое или мультимедийное (фото, видео) содержимое, отправленное в личном или групповом чате. |
| Чат | Закрытое пространство для приватного обмена сообщениями между двумя или более пользователями. |
| Паблик | Открытое сообщество по интересам, куда может вступить любой пользователь для общения и публикации контента. |
| Канал | Личное сообщество человека, состоящее из пользователей, которые им интересуются |
| Подписчик | Пользователь, который подписался на обновления другого пользователя, паблика или канала, чтобы видеть новый контент в своей ленте. |
| Пост | Публикация пользователя или сообщества в ленте, которая может содержать текст, фото, видео и собирать реакции и комментарии. |
| Реакция | Быстрый способ выразить эмоцию к сообщению, посту или комментарию (например, «лайк» или «сердечко»). |
| Комментарий | Текстовый отзыв, оставленный другим пользователем под постом или публикацией. |
| Уведомление | Всплывающее сообщение (push или в приложении), которое информирует пользователя о новом событии: сообщении, запросе в друзья, реакции и т.д. |
| Настройки приватности | Раздел приложения, где пользователь управляет тем, кто и какую информацию из его профиля и активности может видеть. |

## Функциональные требования
Система должна предоставлять следующие функции:

### 1.1. Регистрация нового пользователя

**Форма регистрации:**
| Поле | Тип | Обязательное | Уникальное | Ограничения |
|------|-----|---------------|------------|-------------|
| Телефон | string | Да | Да | В зависимости от кода страны |
| Пароль | string | Да | Нет | Минимум 6 символов из латинского алфавита и цифр, а также спец. символов(! @ # $ % ^ & * ( ) _ + - =) |
| Имя пользователя | string | Да | Да | - |

**Бизнес-правила:**
- Для безопасности аккаунта будет возможность добавления и подтверждения почты.
- Пароль хранится в хэшированном виде.
- Автоматическая активация учетной записи и ее роли.

### 1.2. Аутентификация пользователя

**Форма входа:**
- Номер телефона
- Пароль
- Код, который отослал наш сервис на номер телефона

**Аутентификация:**
- JWT-токен для API-запросов
---

## 2. Управление профилем пользователя

### 2.1. Просмотр профиля

**Основная информация:**
- Имя и фамилия пользователя
- Никнейм
- Город
- Информация о себе
- Возраст


**Базовая статистика:**
- Список и количество друзей и подписчиков пользователя.
- Есть ли собственные каналы или паблики с возможностью скрыть эту информацию.
- Подписки на каналы или паблики с возможностью скрыть эту информацию.

**Бизнес-правила**
- Пользователь может настроить степень приватности своего профиля(доступен только для друзей, никому не доступен и тд).
- Пользователь может не указывать информацию во всех или некоторых полях.
---


## 3. Поиск и добавление в друзья

### 3.1. Поиск пользователей

**Функциональность:**
- Поиск по никнейму пользователя
- Поддержка частичного совпадения

### 3.2. Управление друзьями

**Система друзей:**
- Отправка заявок в друзья
- Принятие/отклонение заявок
- Удаление из друзей

---
## 4. Поиск и добавление в каналы других пользователей

### 4.1. Поиск каналов

**Функциональность:**
- Поиск по названию канала(уникальное)
- Поддержка частичного совпадения

### 4.2. Действия пользователя с другими каналами
- Пользователь отправляет заявку, чтобы добавиться в канал. В случае, если канал приватный, то эту заявку будет рассматривать создатель канала, иначе же пользователь просто зайдет в канал.

---
## 5. Управление собственным каналом 

### 5.1. Основные функции 
- Пользователь может создать канал с заданными характеристиками(название, приватность, описание, аватар)
- Пользователь может выкладывать и удалять публикации из канала(публикации могут включать в себя картинки, гиф-видео, текст, обычные видео)
- Пользователь может удалять подписчиков своего канала или отклонять заявки, в случае, если канал приватный.
- Пользователь может отключать комментарии к своим публикациям.
---

## 6. Дествия пользователя с чужим пабликом 

### 6.1. Основные функции 
- Поиск паблика по названию(уникальное).
- Пользователь может подписаться на соответсвующее сообщество(если оно публичное, то его заявка не рассматривается, иначе же заявку может рассмотреть создатель паблика).
- Пользователь может подключить push-уведомления для новых публикаций паблика.
- Пользователь может комментировать и оценивать публикации паблика
- Пользователь может просмотреть профиль паблика, если он обладает нужными правами приватности. 
- Пользователь может просмотреть подписчиков паблика, если обладает нужными правами приватности.
- Пользователь может посмотреть похожие по теме паблики.
---

## 7. Управление собственным пабликом 

### 7.1. Основные функции 
- Пользователь может создать сообщество с заданными характеристиками(название, приватность, описание, аватар)
- Пользователь может выкладывать и удалять публикации из паблика(публикации могут включать в себя картинки, гиф-видео, текст, обычные видео)
- Пользователь может удалять подписчиков своего паблика или отклонять заявки, в случае, если паблик приватный.
- Пользователь может отключать комментарии и реакции к своим публикациям.
---

## 8. Взаимодествие с картой друзей

### 8.1. Основные функции 
- Пользователь может добавлять людей, которые могут просматривать его геопозицию.
- Пользователь может просмотреть геопозицию друзей в соответвующем разделе, если у него есть соответвующие права доступа.
- Пользователь может узнать о местя для человека, в котором он находится(дома, на работе, спортзал и т.д.), если у него есть соответвующие права доступа.
---

## 9. Сервис по подбору людей со схожими увлечениями

### 9.1. Основные функции 
- Пользователь может посмотреть список людей, с которыми у него схожие интересы и с которыми они близки по геопозиции.
